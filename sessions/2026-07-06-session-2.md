## Decisions Made (do not re-litigate)
- **Stack: Next.js 16 + Tailwind v4 + FastAPI** wrapping the untouched Python engine (chosen over static-Tailwind for declared multi-page growth).
- **Spotify embeds will replace iTunes/Deezer** (plan 004) — feasibility confirmed; resolve track IDs at BUILD time (zero runtime Spotify calls). User wants Spotify's own player (no custom player).
- **Spotify-app chrome:** build inert, clean-room, from scratch (NOT a screenshot backdrop — worse-looking and more legally fraught). Non-clickable is fine for a one-feature concept.
- **`get_artist_by_name` stays exact-only** — `verify_coverage.py` depends on it; fuzzy would corrupt the no-connection metric.
- **Ranking:** match-tier → popularity (once ≥90% enriched, else degree-primary) → degree → name; cross-tier 50× override for prominence-wins.
- **Legal:** clean-room look-and-feel is fine; never Spotify logo/wordmark/assets/font-files/screenshots.
- **Data cleanup (Topsy class) deferred** to its own high-priority data pass.
- Settled & unchanged: MusicBrainz source, Official-only edges, DJ-mix exclusion, depth-3, alias types 1/2.
