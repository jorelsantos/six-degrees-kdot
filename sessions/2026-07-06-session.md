- **Restart the Streamlit preview after any rebuild** — it caches the DB via `@st.cache_resource`, so a running app won't reflect a new graph until restarted.
- **Spotify DB must stay untouched** (`data/collaboration_network.db`) — always verify byte-for-byte after any rebuild; it's the retained fallback.
- **Depth-3 memory:** validated only on 32 GB. A deeper build or a smaller machine could OOM — the noted fallback (SQLite-backed join) is not yet implemented.
- **Deployment blocker looms:** the 179 MB graph is gitignored, so any deploy must solve DB shipping before the app works remotely.
- **Don't over-fix the novelty caveat** — accept-and-report is a settled decision; the song-ordering tweak is a *display-order* improvement, not a re-open of hand-blocklisting.
