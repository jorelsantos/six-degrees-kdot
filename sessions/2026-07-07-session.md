## Decisions Made

- **Deezer guard = exact normalized-name equality**, not `_artist_matches` (bidirectional substring). Substring would accept containment collisions like "June"→"Larry June". A wrong face is worse than the fallback.
- **Persistence is tri-state**: `"none"` sentinel only when the *full* waterfall was consulted and missed (never re-queried); NULL left for transient errors (retryable). This is what makes runtime resolution deploy-tolerable.
- **Persisted URLs are host/scheme-validated** (allowlist: Commons/TheAudioDB/Deezer CDNs, https only) before storing — they're served to every future visitor.
- **U3 (dim photo background) and U4 (motion) were deliberately skipped** this session: U3 risks fighting the plan-009 album-color card; U4 is subjective "feel" the plan flags as off-brand. The DoD permits an explicit skip. **NOTE: the user now wants these RE-OPENED next session (see Next Steps) — U3 as one option, plus a new photo-sizing/centering idea as the alternative.**
- **NOT using Spotify artist images** (ToS gray area) — the Wikidata/TheAudioDB/Deezer sources give broad coverage without it. (Separate from the Spotify *preview* scrape, which remains demo-only.)
- **`requirements.txt` now reflects the real app** (`fastapi`/`uvicorn` in, `streamlit` out).
